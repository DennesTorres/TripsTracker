Operation Flows — Detailed Call Sequences

Add Place
1
AddPlaceForm Frontend
User fills city, country, home checkbox and submits
→ POST/api/places  {cityName, countryIsoAlpha2, isHome}
coordinates are always server-resolved — frontend never sends or receives lat/lon during the add flow
2
PlaceFunctions Functions
Deserializes and validates AddPlaceDto (FluentValidation)
→ PlacesProcess.AddAsync(dto, ct)
3
PlacesProcess.AddAsync — country lookup + geocoding Process
→ CountryBusiness.GetByIsoAlpha2Async(countryIsoAlpha2)
Countries are pre-seeded (195 rows) — lookup always succeeds for a valid ISO code. No creation.
← countryDto (includes countryId)
→ GeocodingBusiness.GeocodeAsync(cityName, countryDto)
→ NominatimGeocodingService.GeocodeAsync → Nominatim /search API
← GeocodingResult {lat, lon, city, stateAbbr, stateName}
├─ if result null → NotFoundException → 404
└─ if result.City ≠ dto.CityName → BusinessRuleException(GEOCODING_MISMATCH) → 409 (UI shows amber prompt)
⚠ geocoding strategy under revision for Nominatim/Photon name mismatch — story #146
4
PlacesProcess.AddAsync — two Business calls Process
→ PlaceBusiness.CreateAsync(place) — persists the new place
→ CountryBusiness.SetVisitedAsync(countryId, true) — always called; Business checks internally
5
PlaceBusiness Business
→ INSERT INTO Places (EF Core)
4b
CountryBusiness.SetVisitedAsync Business
├─ if country already visited: no-op
└─ if not yet visited:
→ UPDATE Countries SET IsVisited = 1 WHERE Id = @id (ExecuteUpdateAsync)
Delete Place
1
PlacesPage Frontend
User clicks delete icon on a place entry
→ DELETE /api/places/{id}
2
PlaceFunctions Functions
Reads {id} from route params
→ PlacesProcess.DeleteAsync(id, ct)
3
PlacesProcess.DeleteAsync Process
→ PlaceBusiness.GetByIdAsync(id) — load place + countryId
→ PlaceBusiness.DeleteAsync(id) → DELETE FROM Places
→ PlaceQuery.GetCountByCountryAsync(countryId) — count remaining
├─ if count == 0 (last place in country):
│ → CountryBusiness.SetVisitedAsync(countryId, false) → UPDATE Countries SET IsVisited = 0
│ ├─ if place was home:
│ │ → CountryBusiness.SetHomeAsync(countryId, false) → UPDATE Countries SET IsHome = 0
│ └─ return DeletePlaceResult { promptHomeCountry: true }
└─ else: return DeletePlaceResult { promptHomeCountry: false }
City Autocomplete
1
AddPlaceForm — city input Frontend
User types in city field → onChange sets cityName
Debounced 300 ms → setDebouncedQuery(cityName)
useCitySuggestions fires when debouncedQuery.length ≥ 2
→ GET /api/cities/suggest?q={query}&country={iso}&limit=5
2
CityFunctions Functions
Reads query params: q, country?, limit
→ GeocodingBusiness.SuggestCitiesAsync(q, limit, countryCode?)
3
GeocodingBusiness → NominatimGeocodingService Business / Integration
→ NominatimGeocodingService.SuggestCitiesAsync(q, limit, countryCode?)
→ Photon API https://photon.komoot.io/api/?q={q}&limit={limit}&lang=en
← GeoJSON FeatureCollection; each feature mapped to CitySuggestion:
│ properties.name → City   properties.countrycode → CountryIsoAlpha2
│ properties.state → StateName   properties.stateabbr → StateAbbr
└─ if countryCode provided: filter results client-side by CountryIsoAlpha2
← CitySuggestion[] returned to frontend
4
AddPlaceForm — selection Frontend
User picks a suggestion from dropdown
selectCitySuggestion(s): sets cityName = s.city, countryIsoAlpha2 = s.countryIsoAlpha2
StateName from suggestion should be displayed in the form after selection — pending story
Read Data — 3 independent processes
Three separate processes, each with its own endpoint, Business class, and DTO. Each serves a different rendering concern. All follow the same straight path: Frontend → Function → Business → Database → DTO response. No branching or conditional calls. AOP interceptors (Logging, Performance, Caching) wrap every Business call.
1
GET /api/places → PlaceDto[] WorldMap · PlacesPage
Purpose: map pins — Lat/Lon position each visited city on the globe
→ PlaceBusiness.GetAllAsync()
→ EF Core: Places JOIN Countries → projects to PlaceDto[]
Includes: Id, City, StateAbbr, StateName, Lat, Lon, CountryId, CountryName, CountryFlag, IsHome
2
GET /api/countries → CountryDto[] CountriesPage · WorldMap
Purpose: country colouring — UserCountries.IsVisited/IsHome determine map fill colour per country (per-user, not global)
→ CountryBusiness.GetAllAsync()
→ EF Core: SELECT * FROM Countries → projects to CountryDto[]
3
GET /api/visited-states → VisitedStateDto[] WorldMap
Purpose: state shading — StateAbbr/CountryId identify which sub-national regions to shade on the map
→ VisitedStateBusiness.GetAllAsync()
→ Raw SQL on VisitedStates SQL VIEW → projects to VisitedStateDto[]
VIEW is derived from Places WHERE StateAbbr IS NOT NULL, grouped by CountryId + StateAbbr — no write path
Update Country — Home flag only (PUT)
1
CountriesPage Frontend
User sets or clears the home flag on a country row
→ PUT /api/countries/{id}/home   ?value=true|false
IsVisited is derived from Places — it is never user-editable
2
CountryFunctions → CountryBusiness Functions / Business
→ CountryBusiness.SetHomeAsync(id, value)
value=true: clears all other home flags, sets this country IsHome=1 (IsVisited unchanged — derived from Places)
value=false: clears IsHome on this country only
← Updated CountryDto

Stage 4 — Multi-User & Profiles

Get / Update User Profile
1
ProfilePage Frontend
User opens profile via avatar dropdown in AppShell nav bar
→ GET /api/me — loads current display name + email
→ PUT /api/me { displayName } — on save
2
UserFunctions Functions
→ UserBusiness.GetByEmailAsync(userContext.Email)
→ UserBusiness.UpdateAsync(userId, UpdateUserDto) — PUT only
(cond) UserContext.UserId populated by JwtValidationMiddleware from JWT sub claim
3
UserBusiness Business
→ SELECT from Users WHERE Email = @email (GET)
→ ExecuteUpdateAsync SET DisplayName = @displayName (PUT)
← UserDto { Id, Email, DisplayName, CreatedAt }
State Border Preference (PUT)
1
CountriesPage Frontend
User toggles the Borders checkbox in the Countries table
→ PUT /api/countries/{id}/borders?value=true|false
(cond) Available countries: those with GeoJSON sub-national boundary data (BR, US today; others via geoBoundaries API — story #170)
2
CountryFunctions → CountryBusiness Functions / Business
→ CountryBusiness.SetShowStateBordersAsync(userId, countryId, value)
→ UPSERT into UserCountries SET ShowStateBorders = @value WHERE UserId = @userId AND CountryId = @countryId
← updated UserCountry row
(cond) Map immediately re-renders state layer for that country on next zoom

Stage 5 — Public Map Sharing

Create / Manage Share Links
1
MapPage — ShareModal Frontend
User clicks Share button → ShareModal opens
→ POST /api/share-links {expiresAt?}
→ GET /api/share-links — list existing links
→ DELETE /api/share-links/{id} — deactivate
2
ShareFunctions → ShareLinkBusiness Functions / Business
→ ShareLinkBusiness.CreateAsync(dto)
token = 32-char URL-safe base64 via RandomNumberGenerator.GetBytes(24)
→ INSERT INTO ShareLinks → ShareLinkDto
→ GetUserLinksAsync() WHERE UserId=current ORDER BY CreatedAt DESC
→ DeactivateAsync(id) SET IsActive=false WHERE Id=id AND UserId=current
View Shared Map — public, no auth
4
SharedMapPage Frontend
Rendered at /#/shared/{token} — unauthenticated hash route
→ GET /api/shared/{token} (no Authorization header)
5
ShareFunctions → PublicMapProcess Functions / Process
JwtValidationMiddleware skips auth (function name "GetSharedMap")
→ PublicMapProcess.GetSharedMapAsync(token)
→ ShareLinkBusiness.GetByTokenAsync(token) — IsActive AND (ExpiresAt IS NULL OR ExpiresAt > now)
(cond) if null → 404
→ IncrementViewCountAsync(token) — ViewCount += 1
→ GetOwnerIdAsync(token) → ownerId
6
PublicMapProcess — cross-user queries Process → Business
→ UserBusiness.GetByIdAsync(ownerId) → display name
→ PlaceBusiness.GetAllForUserAsync(ownerId) (bypasses IUserContext)
→ CountryBusiness.GetAllForUserAsync(ownerId)
→ VisitedStateBusiness.GetAllForUserAsync(ownerId)
← PublicMapDto { OwnerDisplayName, Places, Countries, VisitedStates }
7
SharedMapPage renders Frontend
Header: TripsTracker brand · "DisplayName's travel map"
Stats: N places · N countries · N continents
Read-only WorldMap with owner's places, country fills, state borders
Upload Photo / Add Comment / Rate (Stage 7 UGC)
1
PlaceDetailPanel Frontend
User opens place detail → sees photos, comments, ratings tabs
→ POST /api/places/{id}/photos (multipart upload, ≤10 MB)
→ POST /api/places/{id}/comments or PUT to rate
2
PhotoFunctions / CommentFunctions Functions
→ PlacePhotoBusiness.UploadAsync(placeId, file)
├─ quota check: StorageUsedBytes + fileSize ≤ 100 MB → BusinessRuleException if exceeded
→ BlobStorageService.UploadAsync(path, stream) → Azure Blob / Azurite
→ INSERT INTO PlacePhotos · UPDATE Users SET StorageUsedBytes
Comment flow: INSERT PlaceComments. Rating flow: UPSERT PhotoRatings / CommentRatings.
Points Award / Revoke / Leaderboard (Stage 7 Gamification)
1
PlacesProcess.AddAsync — cascade pioneer scoring Process
After place is saved, calls PointsBusiness.AwardAsync(userId, placeId)
├─ CityAdd (+50): always awarded on new place
├─ CityPioneer (+200): if user is first to add a place in this city (cross-user query)
├─ CountryAdd (+500) / CountryPioneer (+2000): if country newly visited / first visitor
└─ ContinentAdd (+5000) / ContinentPioneer (+20000): same logic per continent
→ INSERT INTO PointEvents (one row per event type) · UPDATE Users SET TotalPoints
2
PlacesProcess.DeleteAsync — revocation Process
→ PointsBusiness.RevokeAsync(userId, placeId) — inserts negative PointEvent rows with OriginalEventId
→ PointsBusiness.ReassignAsync — if user was pioneer, reassigns pioneer event to next eligible user
→ UPDATE Users SET TotalPoints — cache invalidated for affected users
3
AppShell points badge → PointsStatementPanel / Leaderboard Frontend
→ GET /api/me/points → PointsBusiness.GetStatementAsync → statement rows with cityName
→ GET /api/leaderboard → PointsBusiness.GetLeaderboardAsync → top-N by TotalPoints
→ GET /api/users/{id}/points → view another user's statement from leaderboard row
