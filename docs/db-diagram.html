FK Countries → Places is a real foreign key constraint (CountryId NOT NULL). Previous soft reference via CountryName string was replaced in migration 20260321000100.
VIEW VisitedStates is a SQL VIEW — not a table. Derived from Places WHERE StateAbbr IS NOT NULL AND IsDeleted = 0, grouped by CountryId + StateAbbr. No write API — fully derived.
StateAbbr was extracted from the legacy "City(State)" compound field during SchemaCorrections migration. New places receive StateAbbr from Nominatim geocoding at creation time.
All tables implement soft-delete via IsDeleted bit. The VisitedStates view filters IsDeleted = 0 automatically.
Places.Flag was removed — flag emoji is derived from Countries.Flag via the CountryId FK join, not stored redundantly.
