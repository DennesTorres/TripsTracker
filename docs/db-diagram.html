FK Users → Places and Users → UserCountries are real FK constraints (UserId NOT NULL). Every place and country preference is owned by a user — no shared data.
FK Countries → Places is a real foreign key constraint (CountryId NOT NULL). Previous soft reference via CountryName string was replaced in migration 20260321000100.
VIEW VisitedStates is a SQL VIEW — not a table. Derived from Places WHERE StateAbbr IS NOT NULL, grouped by UserId + CountryId + StateAbbr. StateName resolved via MAX(StateName). No write API — fully derived.
UserCountries replaces Countries.IsHome and Countries.IsVisited (removed in Stage 4 migration MultiUserDataIsolationCleanup). IsHome and IsVisited are now per-user, per-country. ShowStateBorders added in AddShowStateBorders migration.
FK Users → ShareLinks is a real FK constraint with cascade delete. Deactivating a link sets IsActive=false; token remains in the table for audit purposes.
FK PlacePhotos and PlaceComments cascade-delete when a Place is deleted. PhotoRatings and CommentRatings cascade-delete when photo/comment is deleted. Blob bytes are deleted separately by application logic before the DB record.
PhotoRatings and CommentRatings use composite PKs (UserId + PhotoId / CommentId) — one rating per user per photo/comment. UPSERT pattern: insert or update.
PlaceComments.ParentCommentId is a nullable self-reference for comment replies (Stage 6 AddParentCommentId migration).
Hard delete — no IsDeleted column. Places deleted with ExecuteDeleteAsync. Cascade deletes: UserCountries rows deleted when User or Country is deleted.
Places.Flag was removed — flag emoji is derived from Countries.Flag via the CountryId FK join, not stored redundantly.
