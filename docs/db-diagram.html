FK Countries → Places is a real foreign key constraint (CountryId NOT NULL). Previous soft reference via CountryName string was replaced in migration 20260321000100.
VIEW VisitedStates is a SQL VIEW — not a table. Derived from Places WHERE StateAbbr IS NOT NULL, grouped by CountryId + StateAbbr. StateName resolved via MAX(StateName). No write API — fully derived.
StateAbbr was extracted from the legacy "City(State)" compound field during SchemaCorrections migration. StateName is the full state name from Nominatim address.state, backfilled for all existing BR and US places.
Hard delete — no IsDeleted column. Places and Countries are deleted with ExecuteDeleteAsync. DeletePlaceProcess manages referential integrity (unsets IsVisited if last place in country).
Places.Flag was removed — flag emoji is derived from Countries.Flag via the CountryId FK join, not stored redundantly.
